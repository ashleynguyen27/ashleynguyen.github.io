Final Project: Progress Report

Ashley Nguyen | Jennifer Zou | Z Wang

Accomplishments

Some things we’ve accomplished so far is learning fluid simulation, understanding the algorithm, and trying out different platforms. We referenced the Macklin and Muller paper which helped us learn a lot. In this process we learned the smooth particle hydrodynamics (SPH) and particle based dynamics or Grid-based and Particle-based frameworks.

To understand the algorithm, we referenced a lot of outside resources for the math heavy parts and understanding some of the physics such as constant density constraints, artificial pressure, vorticity confinement,Jacobi Iteration, Newton's Method, smoothing kernels, etc. We were able to implement the algorithm and data structures in plain code as a model.

In the beginning of our project, we tested out different platforms such as WebGL and starter code from a CMU project. Initially, we tried WebGL. However, we found difficulty implementing the GPU optimization for live rendering and interaction, performing surfacing and rendering. Typically, WebGL incorporates a grid/velocity-field based approach. Currently, we are evaluating whether or not to use the starter code from CMU. Debugging this on the Mac and Linux instructional machines has been proven difficult.

Position-Based Fluid Simulation

One setback for the progress on this project this far, was a lot of things came up and our schedules didn’t match up. So unfortunately, we started a little later than we anticipated.

From the get-go we were very excited to create this fluid simulation. However, we realized we miscalculated the difficulty of the project. We underestimated the amount of work and time that would be needed for this project as we began to look more into it. There are many parts to the project that are each difficult on its own. These include: Particle simulation, surfacing, rendering, shading, and GPU optimization. In addition, there is a lot of technology that we are not familiar with and learning on the go. In addition, we are very limited by our resources we have access to. Most of the resources we are sourcing from is computation fluid dynamics form 184 staff and online references. We found that our online resources are dense and have a variety of approaches.

Another issue we have been struggling with is setting up our environment. Initially, we had planned to use WebGL but because learning it was very time intensive we chose not to. We found starter code from a class project that helped crutch us for basic tasks like camera perspective and shader integration. Finding this alternative was rather time consuming and contributed to our delay.

Work Plan updates

As we move forward, we are going to work on changing a few things. First, considering what the project is asking, we are going to scope down. We are going to focus on understanding and implementing the particle simulation model. To save time, we will also outsource rendering and shading using Blender. Also, we will be using starter code instead of starting from scratch.

From the first proposal, we are changing our approach to mirror CMU’s simulation and build on top of it. We’ve decided to remove the live demo and concentrate more on making sure we have a running simulation. For the simulation, we are constructing the components using cube marching. And always be open for changes.

We are making a plan to split up tasks. However, what’s made it difficult is the structure of the pipeline makes it difficult to work in parallel. We are going to try to make a list of tasks needed to be done and assign a group member to it.
